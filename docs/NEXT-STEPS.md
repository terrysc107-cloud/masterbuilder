# Next Steps

This file tells you what to do immediately after adopting this builder system.

## Recommended Sequence

### Step 1

Choose whether this repo will be:
- the source-of-truth operating system you copy into other repos
- the active working repo for a specific product

### Step 2

If you are starting a new product, read:
- `docs/BOOTSTRAP-A-PROJECT.md`
- `templates/PROJECT-KICKOFF-TEMPLATE.md`

Then create the first project kickoff doc.

### Step 3

Pick the first active phase from `docs/PHASE-MODEL.md`.

For most projects, start with:
- `phases/PHASE-01-product-clarity.md`

If product clarity is already strong, start with:
- `phases/PHASE-02-core-user-flow-clarity.md`

### Step 4

Tell the LLM exactly what to execute.

Example:

```text
Act as Chief Builder. Read docs/MASTER-PLAN.md, docs/MAX-POTENTIAL.md, and phases/PHASE-01-product-clarity.md. Audit the repo first, then execute Phase 01.
```

### Step 5

After each phase:
- record the outcome
- record any debt
- update the memory files
- choose the next phase deliberately
