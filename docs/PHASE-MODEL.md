# Phase Model

This is the reusable phase sequence for building strong apps and websites.

## Phase 01 - Product Clarity

Goal: define the product promise, primary users, primary flows, and what success means.

Lead: `Product Architect`

Support:
- `Scout`
- `Chief Builder`

## Phase 02 - Core User Flow Clarity

Goal: make it obvious where to start, what to do next, and how users progress.

Lead: `Flow Architect`

Support:
- `Scout`
- `Frontend Builder`
- `Design Auditor`
- `Verifier`

## Phase 03 - Frontend Foundation

Goal: establish a strong interface foundation, component structure, layouts, and interaction patterns.

Lead: `Frontend Builder`

Support:
- `Design Auditor`
- `Systems Architect`

## Phase 04 - Core Experience Completion

Goal: make the primary product path fully real end to end.

Lead: `Frontend Builder`

Support:
- `Product Architect`
- `Flow Architect`
- `Verifier`

## Phase 05 - Content / Prompt / Resource System

Goal: make structured assets useful, complete, and organized.

Lead: `Content Systems Builder`

Support:
- `Frontend Builder`
- `Assessment Builder`

## Phase 06 - Assessment / Feedback Layer

Goal: add meaningful checkpoints, measurement, and user guidance.

Lead: `Assessment Builder`

Support:
- `Content Systems Builder`
- `Frontend Builder`
- `Verifier`

## Phase 07 - Offer and Monetization Layer

Goal: connect the product to pricing, premium paths, or services.

Lead: `Offer Strategist`

Support:
- `Flow Architect`
- `Frontend Builder`
- `Design Auditor`

## Phase 08 - Backend and Data Reality

Goal: replace local or fake state with real persistence, auth, and data ownership.

Lead: `Backend Builder`

Support:
- `Systems Architect`
- `Frontend Builder`
- `Verifier`

## Phase 09 - Intelligence and Personalization

Goal: make the product feel more alive, adaptive, and useful based on user behavior.

Lead: `Systems Architect`

Support:
- `Backend Builder`
- `Frontend Builder`
- `Assessment Builder`

## Phase 10 - Production Hardening

Goal: make the product resilient, accessible, and reliable.

Lead: `Verifier`

Support:
- `Backend Builder`
- `Frontend Builder`
- `Design Auditor`

## Phase 11 - Launch Readiness

Goal: make the product shareable, credible, and ready for real users.

Lead: `Chief Builder`

Support:
- `Verifier`
- `Offer Strategist`
- `Flow Architect`
- `Learning Steward`

## Notes

Not every project needs every phase in the exact same depth. The sequence should still be respected unless there is a compelling reason to compress or merge phases.
