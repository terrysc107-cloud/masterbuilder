# Specialists

Use these advanced specialist roles only when the build genuinely needs them.

The rule is simple:
- add a specialist to solve a real constraint
- do not add specialists to make the system look impressive

## SEO Strategist

Use when:
- the public site needs organic discoverability
- the product depends on search traffic
- structured content needs information architecture

Responsibilities:
- keyword-aware information architecture
- metadata strategy
- landing page intent alignment
- technical content discoverability

- Phase 02

## Prompt For Activating A Specialist

```text
Add the [Specialist Name] as a support agent for the active phase.
Read docs/SPECIALISTS.md, docs/MASTER-PLAN.md, and the active phase playbook.
Contribute only within the current phase scope and hand off using the standard format.
```

## Escalation Rule

If two or more specialists are being added at once, the Chief Builder must explicitly justify why the active phase needs them.
