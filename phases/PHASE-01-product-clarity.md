# Phase 01 Playbook - Product Clarity

## Objective

Define the product clearly before building too much of it.

## Lead And Support Agents

- Lead: `Product Architect`
- Support: `Scout`, `Chief Builder`

## Scout Audit Questions

1. What is the product actually trying to help users do?
2. Who is the clearest primary user?
3. What is the strongest user outcome?
4. What features currently distract from the core promise?
5. What should be explicitly delayed?

## Required Outputs

- clear product statement
- clear primary user
- clear primary journey
- project kickoff doc
- phase contract for the next phase

## Exit Criteria

- the product can be described in a few clear sentences
- the primary user and outcome are no longer fuzzy
- the next phase can begin without strategic confusion

## Execution Prompt

```text
Execute Phase 01 using docs/MASTER-PLAN.md and this playbook.
Adopt the roles of Product Architect, Scout, and Chief Builder.
Audit first, then clarify the product without drifting into implementation-heavy work.
```
