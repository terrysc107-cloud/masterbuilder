# Phase 02 Playbook - Core User Flow Clarity

## Objective

Make it obvious where users start, what they do next, and how they progress.

## Lead And Support Agents

- Lead: `Flow Architect`
- Support: `Scout`, `Frontend Builder`, `Design Auditor`, `Verifier`

## Scout Audit Questions

1. What is the single clearest first action today?
2. What is the single clearest next action for returning users?
3. Which screens have too many competing calls to action?
4. Is resume behavior obvious?
5. Is completion language clear?

## Required Outputs

- stronger onboarding path
- clearer CTA hierarchy
- stronger resume and continue logic
- directive navigation and progress cues

## Exit Criteria

- first-time users know where to start quickly
- returning users know what to do next quickly
- the product stops feeling directionally vague

## Execution Prompt

```text
Execute Phase 02 using docs/MASTER-PLAN.md and this playbook.
Adopt the roles of Flow Architect, Scout, Frontend Builder, Design Auditor, and Verifier.
Audit first, then improve the user journey without widening scope.
```
