# Phase 03 Playbook - Frontend Foundation

## Objective

Establish the UI structure, component patterns, and interaction quality needed for a durable product.

## Lead And Support Agents

- Lead: `Frontend Builder`
- Support: `Design Auditor`, `Systems Architect`

## Scout Audit Questions

1. Is the component structure coherent?
2. Are interaction patterns consistent?
3. Is the visual language intentional?
4. Are responsive layouts strong enough?
5. What frontend debt will slow later phases?

## Required Outputs

- strong component patterns
- cleaner route and layout structure
- intentional responsive UI
- fewer fragile frontend inconsistencies

## Exit Criteria

- the frontend feels structurally sound
- new features can build on reusable patterns
- design quality is improving, not drifting

## Execution Prompt

```text
Execute Phase 03 using docs/MASTER-PLAN.md and this playbook.
Adopt the roles of Frontend Builder, Design Auditor, Systems Architect, and Scout.
Audit the UI foundation first, then strengthen the frontend structure.
```
