# Phase 04 Playbook - Core Experience Completion

## Objective

Make the product's primary path fully real end to end.

## Lead And Support Agents

- Lead: `Frontend Builder`
- Support: `Product Architect`, `Flow Architect`, `Verifier`

## Scout Audit Questions

1. What is the app's primary promise?
2. Which parts of the main experience are still partial?
3. Where does the user journey feel fake, shallow, or incomplete?
4. Which pieces block a credible demo?
5. What should not be touched yet?

## Required Outputs

- one strong complete core path
- reduced scaffolding in the main journey
- higher confidence that the product does something real

## Exit Criteria

- the main path can be used start to finish
- the product feels materially more real
- major holes in the core experience are closed

## Execution Prompt

```text
Execute Phase 04 using docs/MASTER-PLAN.md and this playbook.
Adopt the roles of Frontend Builder, Product Architect, Flow Architect, Verifier, and Scout.
Audit the main product path first, then complete it end to end.
```
