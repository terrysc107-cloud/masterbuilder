# Phase 09 Playbook - Intelligence And Personalization

## Objective

Make the product feel more adaptive, useful, and context-aware over time.

## Lead And Support Agents

- Lead: `Systems Architect`
- Support: `Backend Builder`, `Frontend Builder`, `Assessment Builder`

## Scout Audit Questions

1. What user context is already available?
2. What recommendations or personalization are already happening?
3. What would make the product feel more alive?
4. Which adaptive behaviors are truly useful?
5. What should be rule-based first before becoming more advanced?

## Required Outputs

- smarter next-step logic
- more useful personalization
- stronger context-aware surfaces
- clearer adaptive system boundaries

## Exit Criteria

- the product feels more tailored
- recommendations feel useful, not random
- personalization adds value without excess complexity

## Execution Prompt

```text
Execute Phase 09 using docs/MASTER-PLAN.md and this playbook.
Adopt the roles of Systems Architect, Backend Builder, Frontend Builder, Assessment Builder, and Scout.
Audit the current intelligence layer first, then improve personalization in a grounded way.
```
